# Glossary

Use this glossary in user-facing communication. Prefer the Chinese alias in normal conversation, and include the English term in parentheses when precision helps.

## Operation Names

| Simple command | English term | 中文解释 |
| --- | --- | --- |
| `启动` | `bootstrap` | 为新项目、新阶段建立最小控制面 |
| `规划` | `plan` | 明确目标、约束、架构、路线、测试与执行切片 |
| `执行` | `execute` | 按当前切片实现，并同步更新控制面 |
| `继续` | `continue / resume` | 恢复当前状态并继续当前执行线 |
| `进展` | `progress` | 汇总项目当前进展、风险、下一步 |
| `整改` | `retrofit` | 把已有项目调整到这套治理规范 |
| `收口` | `closeout` | 收尾当前切片或阶段，并准备下一阶段 |

## Common Compound Terms

| English term | 中文解释 |
| --- | --- |
| `retrofit audit` | `整改审计`，先检查现状并给出整改方案，不直接改文件 |
| `guided retrofit` | `引导式整改`，先审计和确认，再实施整改 |
| `direct retrofit` | `直接整改`，按最小安全改动直接落地 |
| `control surface` | `控制面`，指 `brief / plan / status` 这类当前执行真相来源 |
| `living document` | `活文档`，执行过程中持续更新的文档 |
| `durable document` | `稳定文档`，长期有效、不会频繁变动的参考文档 |
| `generated evidence` | `生成证据`，如测试、评估、审计报告等运行产物 |
| `delivery tier` | `交付层级`，即 `small / medium / large` |
| `active slice` | `当前切片`，当前正在执行的最小工作单元 |
| `phase closeout` | `阶段收口`，确认当前阶段完成并定义下阶段入口 |
| `progress dashboard` | `进展面板`，用于汇总全局、子项目、证据和下一步的简洁状态视图 |
| `module dashboard` | `模块面板`，用于汇总大项目各模块当前状态、已有能力和下一步 |
| `module layer` | `模块层控制面`，指 `.codex/module-dashboard.md` 与 `.codex/modules/*.md` |

## User-Facing Rule

When speaking to the user:

1. prefer the Chinese alias first
2. add the English term in parentheses on first mention when useful
3. avoid unexplained English-only workflow terms
